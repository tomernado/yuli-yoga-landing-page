## Why This Resource

This resource should provide structured guidance for identifying, analyzing and resolving performance bottlenecks across frontend, backend and infrastructure.

Prefer established performance tools over ad-hoc optimization techniques.

---

## Use When

Use this resource for tasks involving:

- Performance optimization.
- Performance profiling.
- Slow application behavior.
- Slow page loads.
- Large bundle analysis.
- API latency.
- Database performance.
- Memory usage.
- CPU optimization.
- Production performance validation.

---

## Do Not Use When

Do not use this resource for:

- Routine feature development.
- Small UI changes.
- Documentation.
- General refactoring.
- Non-performance related tasks.
